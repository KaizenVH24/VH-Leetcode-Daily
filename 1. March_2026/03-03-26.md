# Intuition
<!-- Describe your first thoughts on how to solve this problem. -->
The string `Sn` is built recursively:

- The left half is exactly `S(n-1)`
- The middle element is always `"1"`
- The right half is the reverse of `S(n-1)` with all bits inverted

This structure creates symmetry.

The full string length is `2^n - 1`.  
The middle position is always at `2^(n-1)`.

Instead of building the entire string (which grows exponentially), we can use this symmetry:

- If `k` is at the middle → the answer is `"1"`.
- If `k` is in the left half → the answer is the same as in `S(n-1)`.
- If `k` is in the right half → mirror the position to the left half and invert the result.

By repeatedly shrinking the problem from `Sn` to `S(n-1)`, we avoid constructing large strings.

# Approach
<!-- Describe your approach to solving the problem. -->
1. While `n > 1`:
   - Compute total length: `(1 << n) - 1`
   - Compute middle index.
2. If `k` equals the middle → return `"1"` (adjusting if inverted).
3. If `k` is in the right half:
   - Reflect `k` to its mirrored index.
   - Toggle an `invert` flag.
4. Reduce `n` and continue.

When `n` becomes 1, the base string is `"0"`.  
Return `"0"` or `"1"` depending on whether the inversion flag is set.

This avoids recursion and avoids building the string entirely.


# Complexity

- **Time complexity:**  
  O(n)  
  We decrease `n` one level at a time.

- **Space complexity:**  
  O(1)  
  Only a few variables are maintained; no recursion stack or large string storage.

# Code
```python3 []
class Solution:
    def findKthBit(self, n: int, k: int) -> str:

        invert = False
        
        while n > 1:
            length = (1 << n) - 1
            mid = (length // 2) + 1
            
            if k == mid:
                return "0" if invert else "1"
            
            if k > mid:
                k = length - k + 1
                invert = not invert
            
            n -= 1
        
        return "1" if invert else "0"
```
