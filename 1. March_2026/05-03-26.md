### 1758. Minimum Changes To Make Alternating Binary String

# Intuition

An alternating binary string cannot have two identical characters next to each other.  
If we think about how such strings look, there are only two valid possibilities for any length:

- One that starts with `0`: `010101...`
- One that starts with `1`: `101010...`

So instead of trying to modify the string in many different ways, we can simply compare the given string with these two possible patterns and count how many characters would need to change in each case.

The answer will be the smaller of those two counts.

---

# Approach

We maintain two counters:

- `changes_start_0` → number of changes needed if the string should follow the pattern starting with `0`
- `changes_start_1` → number of changes needed if the string should follow the pattern starting with `1`

We iterate through the string once and check what the expected character should be at each index for both patterns.

- For the pattern starting with `0`, the expected value at index `i` is `i % 2`
- For the pattern starting with `1`, the expected value at index `i` is `(i + 1) % 2`

If the character in the string does not match the expected value, we increment the corresponding counter.

Finally, we return the minimum of the two counts since we want the fewest changes.

---

# Complexity

- **Time complexity:**  
  $$O(n)$$  
  We scan the string once.

- **Space complexity:**  
  $$O(1)$$  
  Only two counters are used.

---

# Code
```python
class Solution:
    def minOperations(self, s: str) -> int:

        changes_start_0 = 0
        changes_start_1 = 0
        
        for i, ch in enumerate(s):
            if ch != str(i % 2):
                changes_start_0 += 1
            
            if ch != str((i + 1) % 2):
                changes_start_1 += 1
        
        return min(changes_start_0, changes_start_1)
```
