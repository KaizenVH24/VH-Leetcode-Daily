# 1689. Partitioning Into Minimum Number Of Deci-Binary Numbers

---

## Intuition

A deci-binary number contains only `0` and `1` in its digits.

When multiple deci-binary numbers are added together, each digit position in the final sum is simply counting how many `1`s were placed in that column.

So if a digit in `n` is `7`, that means at least **7 different deci-binary numbers** must contribute a `1` at that position.

From this observation:

> The minimum number of deci-binary numbers required is equal to the **maximum digit** in the string.

Everything else is just different ways to reach that conclusion.

---

# 1. Brute Force Approach

### Idea

Simulate the process literally:

- While the number is not zero:
  - Create one deci-binary number by placing `1` wherever the digit is non-zero.
  - Subtract 1 from all non-zero digits.
  - Count how many times we do this.

This builds the answer layer by layer.

### Code

```python
class Solution:
    def minPartitions(self, n: str) -> int:
        digits = list(map(int, n))
        operations = 0

        while any(digits):
            for i in range(len(digits)):
                if digits[i] > 0:
                    digits[i] -= 1
            operations += 1

        return operations
```

### Complexity

- Time: O(n × max_digit)
- Space: O(n)

Not ideal. If the maximum digit is 9 and the length is 10⁵, this becomes inefficient.

--- 

# 2. Optimized Approach

### Idea

Instead of subtracting layer by layer, just notice:
- Each subtraction layer corresponds to one deci-binary number. So the total number of layers equals the largest digit in the string. We can compute the maximum digit in one pass.

## Code

```python
class Solution:
    def minPartitions(self, n: str) -> int:
        max_digit = 0
        for ch in n:
            max_digit = max(max_digit, int(ch))
        return max_digit
```

### Complexity

- Time: O(n)
- Space: O(1)

--- 


# 3. Most Optimized Approach

### Idea

Python already compares digit characters correctly ('0' < '1' < ... < '9'). So we don’t even need to convert inside the loop. Just take the maximum character and convert once.

### Code

```python
class Solution:
    def minPartitions(self, n: str) -> int:
        return int(max(n))
```

### Complexity

- Time: O(n)
- Space: O(1)
